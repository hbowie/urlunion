Boeing Docs Normalization

PSTextMerge is designed to perform some special processing specifically customized for Boeing in Mesa, Arizona.

This special processing is packaged into a "normalization" routine.

Implementation Considerations

In order to invoke this processing, some special actions must be taken prior to executing PSTextMerge.

Turn Normalization On

Before normalization is even available as an option, a run-time flag must be set to indicate to PSTextMerge that it is running in a Boeing environment. Previous versions of the program have used different mechanisms for doing this, but the latest versions now look for a file named 'boeing.txt' within the same folder as the PSTextMerge executable. The contents of the file are irrelevant.

(Note that normalization can also be enabled by setting 'normalization' to 'true' within User Preferences, but PSTextMerge offers no way to modify this setting, and the actual storage of user preferences vary by platform: for Windows, they are stored in the Registry.)

Create Lookup Tables

The Boeing Normalization routine will look for two different lookup tables.

  • gov_doc_types.txt
  • sys_doc_types.txt

Each of these tables must contain three different columns.

Code
The code being looked up. This will be a code extracted from a file name.
Meaning
The definition of this code.
Clarification
Optional clarifying comments about this code.

The lookup tables may be stored in the execution folder or the home folder. The execution folder is the folder containing the PSTextMerge executable. The home folder is the folder containing the script being executed, if a script is being executed.

Output Fields

The following columns will be output from the Boeing normalization routine.

System ID
The System identifier (A400, IEDB, etc.)
Change Number
The Change Number embedded in the file or folder name.
Sub-Folder
Name of any sub-directory within the System or Change Number folder.
Stage
The stage to which the file has progressed: development, review or released.
File Name
The name of the file.
Path
The complete path to, but not including, the file name.
Doc Type
The document type, as extracted from the file name.
Doc Type Desc
A brief description of the document type, accessed from a lookup table.
Last Mod Date
Time of last change to the file, in "hh:mm:ss zzz" format, where "hh" is a 24-hour (military) hour, "mm" is minutes, "ss" is seconds and "zzz" is an abbreviation of the time zone.
Errors
Any errors discovered in the format of the folder structure or file name.
